user test results.

user I - tim

With this current iteration, when the user hovers over the color in the palette, the background color of the section temporarily changes using the 'mouseover' event listener, and changes back to the default setting after the the mouse leaves the color.

However, Tim suggested that when the user clicks on the color in the palettes, the background color of the section changes permanently. This is to offer a more fixed view of the portfolio images against the palette color, as well as maximize the interaction on the webpage. I will certainly be making this change in my final iteration!

user II - peter

With this current iteration, the portfolio images displayed cannot be interacted with by the user. Personally, I had a couple ideas on user interaction with the images, but did not have any I was confident in being able to do with the limited time I have during Week 10! Currently, the images have a dark-gray drop-shadow.

However, Peter suggested that when the user clicks on the image, the drop-shadow color of the image changes to one that will offer maximum contrast to create a 'glow.' I will certainly be making this change as it plays well with my theme of bold colors and contrast!

user III - annette

With this current iteration, there are no 'close' buttons for pop-ups & overlays, which Annette noticed and immediately notified me about. Moreover, currently, the images have no user-based behavior.

Annette suggested that when the user hovers over the image, another image will pop-up related to the person the image features. For instance, for the first section on the Indian Classical Dance Team at UC Davis, when the user hovers over an image, another image from a stage performance will replace the original image. This would add a warm narrative, which I love and will certainly implement!
